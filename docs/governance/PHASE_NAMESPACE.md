## Governance program

- `GOV-A` — Source of truth
- `GOV-B` — Pull-request organization
- `GOV-C` — Registry and ownership
- `GOV-D` — GitHub enforcement
- `GOV-E` — CI normalization
- `GOV-F` — Attack-surface reduction
- `GOV-G` — Product-readiness review
- `GOV-H` — Final executive decision

## Product work

Product work must use descriptive identifiers and must never use a bare numeric phase:

- `PRODUCT-SEO-READINESS`
- `PRODUCT-CONTENT-BRAIN-VALIDATION`
- `PRODUCT-CRM-CONTROLLED-WRITES`

A governance stage does not authorize product execution. Product scope, evidence, approvals, and rollback remain separate.
