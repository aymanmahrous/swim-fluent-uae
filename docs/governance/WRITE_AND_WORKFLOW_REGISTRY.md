## Rules

- This registry is based on repository source review only and authorizes no execution.
- `Current` means a path exists in reviewed source/history; it does not prove live configuration.
- `Blocked` means no merge, dispatch, deployment, migration, provider call, write, generation, publishing, or Production action is authorized.
- Production-write workflows must remain manual-only, target an approved SHA/ref, use protected environments, least-privilege secrets, concurrency control, independent approval, and post-run read-only verification.
- Named individuals are not yet assigned; accountable role names keep activation blocked until GOV-D or a separately approved ownership record.

## Kill switches

| Sensitive path | Stop method | Kill switch owner | After stop |
|---|---|---|---|
| Public booking writes | disable affected release/route through approved deployment control or server gate; never expose DB secret | Booking Product Owner | preserve existing bookings, reject new requests safely, publish operational notice only if approved |
| Staff booking/CRM/inbox writes | disable mutation routes/UI or revoke affected role permission | Staff/Booking/CRM Operations Owner | retain read access, reconcile final successful receipt, investigate in-flight requests |
| Content edits/transitions | disable mutation and scheduling controls | Content Operations Owner | items remain readable and in last confirmed state; no publishing |
| AI Content Brain | keep PR #170 frozen; disable AI environment/model secret/feature gate | AI Operations Owner | reject new runs, cancel queued work if supported, retain run and audit records |
| Database migrations | do not dispatch/apply; lock protected environment/remove operator access | Database Owner | read-only inventory, incident review, choose forward-fix/restore plan |
| Media worker/generation | disable worker/cron/environment secret | AI Media Owner | stop claiming jobs, preserve queue state, reconcile provider jobs and orphaned assets |
| Publishing/scheduler | disable scheduler/worker and revoke scoped provider token | Publishing Owner | halt new publications, reconcile ambiguous receipts, correct/unpublish only with approval |
| Storage mutation | revoke/disable storage write credential or worker | Media Operations Owner | reads continue where safe; quarantine suspect objects and compare hashes |
| Production workflow | disable protected environment approvals or workflow access | Release Owner | no new dispatch; preserve run history; perform read-only status review |
| Governance writes | stop branch commits | Governance Owner | compare branch to `main`; revert only by new auditable commit |

## Rollback evidence standard

Each rollback must record: repository and path, owner/operator, independent approval, target environment, target SHA or release, before-state, after-state, request/job/migration/publication IDs, timestamps, logs/receipts, verification method, unresolved side effects, and final decision. History rewriting and undocumented direct database correction are prohibited.
